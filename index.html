排序

December 14, 2017

1. 插入排序

插入排序就好像我们打扑克排序一样，一手乱牌，从第二张开始抽取，插入到合适的位置(后面的一张牌大于抽出的牌，前面的牌小于或等于抽取的牌)大致原理是这样.

插入排序也是一样，从第二个元素开始，从前往后对比，找到合适位置的下标，将目标元素插入到该处即可

2. 希尔排序

希尔排序是对插入排序的优化，有一个增量d的概念，增量从大到小直至为1,比较是以一个元素与另一个相距d的元素比较，arr[i]>arr[i+d],则交换位置(升序),方便记忆的话3个for+1个if,希尔排序重在比较。

3. 选择排序

  • 简单选择排序

    选择排序理解起来比较简单，就是找到最小的(升序),和第一个元素交换位置，然后找到第二小的和第二个元素交换位置，依次类推，直到最后一个元素和倒数第二的元素处理

  • 二元选择排序

    可能你会说，既然都是要查找的，为什么不在一次循环中直接找出最大值和最小值呢？接下来就是介绍该思想对应的排序–二元选择排序。这是对简单选择排序的优化,就是在一次遍历的时候选出最大值和最小值，分别在数组的前面和后面,从而减少循环的次数.

Postman请求中的各个参数的意思

October 17, 2017

简介

postman中POST请求的body中有form-data、x-www-form-urlencoded、raw、binary四种选项，其对应HTML的enctype 属性,规定在发送到服务器之前应该如何对表单数据进行编码。
表单数据提交。

1. form-data

http请求中的multipart/form-data，会将表单数据处理为一条消息，以标签为单元，用分隔符分开。可以上传键值对或者文件，需要上传文件时需要设置key（键）的后面为file。

由于有boundry隔离，所以该类型既可以上传文件，也可以上传键值对，并可以上传多个文件。

2. x-www-form-urlencoded

等价于application/x-www-form-urlencoded，会将表单数据转为键值对，可以在servlet直接用getParameter获取指定参数。但是只能传text类型的值。

3. raw

自由度最高一种方式，在这里可以设置需要传递的参数的类型，比如文件，字符串，json形式，功能最为全面。可设置类型如下:

4. binary

相当于application／octet-stream，只能上传二进制文件，通常用来上传文件，且一次只能上传一个文件。

总结

一般在使用中都是使用form-data，功能基本够用，既可上传文件也可上传文本。但是form-data与x-www.form-urlencoded的区别就是前者可以上传文件，且最后转化成一条消息（推测为JSON，暂未测试），而后者只能上传文本类型。

socket学习

October 14, 2017

socket

简介

又叫套接字，通信的基石，它是对TCP/IP协议的封装，本身并不是协议，而是一个调用接口（API），通过Socket，我们才能使用TCP/IP协议，socket支持TCP／IP协议的网络通信的基本单元，包含网络通信的必须的5种信息：

  1. 连接使用的协议
  2. 本地主机的IP地址
  3. 本地进程的协议端口
  4. 远地主机的IP地址
  5. 远程进程的协议端口

多个TCP连接或多个应用程序进程可能需要同一个TCP协议端口传输数据。
为了区别不同的应用程序进程和连接，操作系统为应用程序与
TCP／IP协议交互提供了套接字（socket）接口。应用层可以和传输层通过socket接口，区分来自不同应用程序进程或网络连接的通信，实现数据传输的并发服务。

socket连接至少需要一对套接字，运行于客户端（ClientSocket）和服务端（ServerSocket），连接分3个步骤：

  1. 服务器监听
  2. 客户端请求
  3. 连接确认

socket支持不同的传输层协议（TCP，UDP），使用TCP协议连接就是一个TCP连接，UDP同理。
